# Creación de nuevas incidencias en el proyecto NVDA

Esta página se ha creado para servir como explicación al rellenar la la plantilla de incidencias de NVDA en GitHub (en inglés).

La plantilla usa markdown de Github, para proporcionar formato en los encabezados, listas, citas, etc. Si no estás familiarizado con esto, tómate algo de tiempo para aprender sobre el markdown de Github (en inglés)

Advertencia: salvo en raras circunstancias, es necesario completar toda la plantilla. Probablemente se cerrará tu incidencia si no se ha seguido esta plantilla.

Adjuntos e imágenes

Es importante incluir cualquier archivo necesario para reproducir la incidencia. Este podría ser un archivo necesario para una suite ofimática, o un enlace a un recreador de código como CodePen o JSFiddle, o tal vez un archivo HTML independiente. Github no permite adjuntar todos los tipos de archivos. Sin embargo, sí se permiten los archivos zip, por lo que puedes comprimir tu archivo y adjuntar el resultado.

Registros

En la mayor parte de los casos, un archivo de registro de NVDA es increíblemente útil cuando se intenta comprender o resolver un problema, así que no olvides adjuntarlo. Hay instrucciones al respecto en los documentos para desarrolladores de esta web. Si lo que obtienes es un archivo de minivolcado (nvda_crash.dmp), inclúyelo también.

Capturas de pantalla

Aunque las imágenes / capturas de pantalla son bienvenidas para ayudar a explicar un problema, ten en cuenta que muchos de los desarrolladores de NVDA son ciegos y apreciarán mejor la imagen si está descrita como texto. La mayoría de las herramientas te permiten copiar texto fuera de ellas.

Consejos

  • Plantéate ir a la pestaña Preview para leer tu incidencia en cualquier momento y confirmar que queda como esperas.
  • Si tienes problemas siguiendo esta plantilla, o con la investigación inicial necesaria, pide educadamente asistencia a la fantástica comunidad que se encuentra en la lista de correo de NVDA en español

La plantilla

Al principio de la plantilla hay un bloque de comentarios en HTML que apunta hacia la versión inglesa de esta página en la wiki de NVDA. Se puede dejar como está, y no aparecerá una vez que se guarde la incidencia. Elimínalo si lo deseas.

Después del comentario en HTML, puedes proporcionar una breve descripción de la incidencia. A continuación, rellena cada una de las siguientes secciones:

Pasos para reproducir:

Esta sección debería contener una lista de los pasos que realizas para demostrar el problema. Mientras los estés realizando, haz una copia de tu registro de NVDA para adjuntarla a la incidencia.

Ejemplo:

  1. Abrir Chrome
  2. Navegar a www.google.com
  3. Escribir "Hola"
  4. Pulsar intro

Comportamiento actual:

Esta sección debería explicar qué sucede realmente al llevar a cabo los pasos anteriores.

Ejemplo:

Se oye un sonido de error de NVDA.

Comportamiento esperado:

En esta sección deberías explicar qué esperas que pase al seguir los pasos anteriores.

Ejemplo:

No debería haber sonido de error, y la página debería cambiar para mostrar los resultados de la búsqueda.

Configuración del sistema:

Esta sección tiene varias subsecciones. Estas van sobre el sistema que estás usando al encontrar el fallo. Si encuentras el fallo con diversas configuraciones diferentes, lístalas. Esto ayuda a los desarrolladores a reproducir el problema, o investigar por qué el problema les ocurre a algunos usuarios y a otros no.

NVDA Instalado, portable o ejecutándose desde el código fuente:

Esto indica a los desarrolladores cómo estás utilizando NVDA.

Ejemplo:

Instalado

Versión de NVDA:

La versión de NVDA que estás utilizando para demostrar el problema. Se puede copiar desde la opción Acerca de, en el menú Ayuda de NVDA.

Ejemplo:

next-15370,21fd217a

Versión de Windows:

La versión de Windows que estás usando. Puedes copiarla desde la aplicación "Información del sistema":

  1. Pulsa Windows+r para abrir el diálogo ejecutar
  2. Escribe msinfo32 y pulsa intro
  3. Desde el elemento de árbol "Resumen del sistema", pulsa tab para llegar al panel derecho.
  4. Pulsa flecha abajo dos veces para llegar a "Versión"
  5. Pulsa control+c para copiar la información de versión.

Ejemplo:

Windows 10 Versión 1607, Compilación 14393.1066

Nombre y versión de otro software en uso al reproducir el problema:

Normalmente puedes obtener información desde el diálogo "Acerca de" del software.

Ejemplo:

Chrome versión: 67.0.3396.87

Otra información de tu sistema:

Se trata de otra información sobre tu configuración que crees que podría ser importante para los desarrolladores.

Ejemplo:

Ejecuto Windows 10 desde una máquina virtual

Otras preguntas:

Esta sección tiene subsecciones que preguntan sobre los pasos básicos de la investigación que puedes haber completado. Si quieres, añade más información aquí para contarle a los desarrolladores qué pruebas has hecho.

¿El problema todavía ocurre después de reiniciar tu PC?

Reiniciar tu ordenador ayudará a asegurar que el software se encuentra en un estado más fresco.

¿Has probado otras versiones de NVDA?

Saber si el problema ocurre en versiones anteriores ayuda a los desarrolladores a entender si es un problema recién introducido, y les da una pista sobre si es un cambio en NVDA o el responsable del problema es otro programa.

Ejemplo:

  • NVDA 2018.1 - no hay sonido de error
  • NVDA 2018.2 - tiene el sonido de error

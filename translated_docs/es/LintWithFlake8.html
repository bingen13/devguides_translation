# Pruebas de conformidad con Flake8

Pruebas de conformidad con Flake8

Introducción

Durante muchos años, NV Access y la comunidad de NVDA han recibido cientos de solicitudes de cambio de muchos colaboradores, incluyendo nuevas características, corrección de errores, y traducciones. Mientras que muchos debates de las solicitudes de cambio se centraban en la implementación, había varias que necesitaban editarse para hacer que el nuevo código cumpliera con las pautas de código de NVDA.

En 2019, como parte de un gran esfuerzo para migrar NVDA a Python 3 y facilitar el proceso de revisión de las solicitudes de cambio, NV Access introdujo herramientas de pruebas de conformidad, específicamente Flake8. Las pruebas de conformidad permiten que el código fuente se compruebe frente a unas pautas de codificación y da una oportunidad para que los colaboradores corrijan errores antes de enviar cambios para que sean revisados por la comunidad.

Para obtener ayuda con estándares de código o trampas con los mensajes de comprobación de conformidad, visita: - https://github.com/nvaccess/nvda/wiki/Contributing - tests/lint/readme.md

En la mayoría de los archivos del código fuente, cubiertos por la licencia GNU Gpl, usa una cabecera como la siguiente:

Reemplaza: - <Tu nombre> por tu nombre. - <Año de creación> por el año de creación del archivo - <Año de última actualización> por el año en que se actualizó el archivo por última vez. En los archivos nuevos no es necesario. Por ejemplo, un archivo que se ha creado en 2018 podría tener #Copyright (C) 2018

# A part of NonVisual Desktop Access (NVDA)
# Copyright (C) <AÑO DE CREACIÓN>-<AÑO DE ÚLTIMA ACTUALIZACIÓN> NV Access Limited, <TU NOMBRE>
# This file may be used under the terms of the GNU General Public License, version 2 or later.
# For more details see: https://www.gnu.org/licenses/gpl-2.0.html

Algunos archivos están cubiertos por la licencia GNU LGPL, como los ejemplos o el cliente del controlador utilizado por muchos programas para comunicarse con NVDA. Esto permite que cualquiera use todo o parte de su contenido tal y como está. En este caso utiliza la siguiente cabecera:

# A part of NonVisual Desktop Access (NVDA)
# Copyright (C) <AÑO DE CREACIÓN>-<AÑO DE ÚLTIMA ACTUALIZACIÓN> NV Access Limited, <TU NOMBRE>
# This file may be used under the terms of the GNU Lesser General Public License, version 2.1.
# For more details see: https://www.gnu.org/licenses/lgpl-2.1.html

En algunos archivos se usa un estilo más antiguo para referirse a las personas que han contribuido con él. Los contribuyentes no se expresan como una lista de nombres, sino escribiendo algo como: Copyright (C) 2006-2016 NVDA Contributors. NV Access trabaja para eliminar este viejo estilo y utilizar uno más nuevo indicando todos los nombres. El proceso es un poco complicado. Hay que mirar los registros de Git para ver cuándo se creó el archivo y quién lo ha tocado, y construir una línea de copyright adecuadamente.

Nota: NV Access decidió borrar el nombre de los archivos que se podía ver en la primera línea del comentario de copyright, ya que no aporta nada y es una causa frecuente de errores al renombrar o copiar las cabeceras de un archivo a otro.
